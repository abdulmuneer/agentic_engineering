# Frontend Engineer

## Purpose

Builds the user-facing application experience based on product requirements and UX/UI design.

## Core Responsibilities

- Implement screens, components, client-side state, routing, and user interactions.
- Integrate frontend workflows with backend APIs.
- Ensure responsive behavior across supported devices and browsers.
- Implement accessibility requirements.
- Handle loading, error, empty, and permission states.
- Write frontend unit, component, and end-to-end test support where appropriate.

## Key Deliverables

- User interface implementation.
- Reusable frontend components.
- API integration code.
- Frontend test coverage.
- Accessibility and responsive behavior fixes.

## Review Responsibilities

- Performs peer review for frontend-related changes when assigned.
- Responds to review findings from the Code Quality Reviewer, UX/UI Designer, QA Engineer, and Security Reviewer.

## Document Update Responsibilities

| Document | When This Role Updates It |
|---|---|
| `output/repositories` | Continuously during implementation when frontend code, tests, components, routes, state, styling, or configuration change. |
| `output/documentation` | When frontend behavior, component usage, accessibility notes, user-facing states, or developer setup details need documentation. |
| `program/trackers/backlog_tracker.md` | When frontend work is started, blocked, split, completed, or when new UI defects or technical debt are found. |
| `program/trackers/raid_log.md` | When browser compatibility, accessibility, performance, UX feasibility, or integration risks are discovered. |
| `program/trackers/dependency_tracker.md` | When frontend work depends on designs, API contracts, assets, content, feature flags, or environment configuration. |
| `program/trackers/test_readiness_tracker.md` | Before QA handoff when frontend test coverage, test data, supported states, or environment readiness changes. |
| `learnings/technical_discovery_template.md` | When implementation reveals a UI, accessibility, browser, or state-management lesson. |

## Collaborates With

- UX/UI Designer for design fidelity and usability.
- Backend Engineer for API contracts.
- Integration Engineer for cross-system workflows.
- QA Engineer for acceptance testing.
- Documentation & Customer Feedback Owner for user-facing copy and release notes.

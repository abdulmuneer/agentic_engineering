# Sprints

This folder lists and stores sprint-specific planning and execution records.

Use `sprint_index.md` to list all sprints. For each sprint, create a folder or file using a consistent name such as `sprint_001`, `sprint_002`, or `sprint_YYYY_MM_DD`.

## Recommended Sprint Folder Contents

| File | Purpose |
|---|---|
| `plan.md` | Sprint goal, committed work, capacity notes, and responsibilities. |
| `execution_notes.md` | Blockers, changes, decisions, and mid-sprint updates. |
| `demo_notes.md` | Demo agenda, feedback, accepted work, and follow-up. |
| `retrospective.md` | What worked, what did not, and improvement actions. |

## Rules

- Keep sprint-specific records here.
- Keep reusable tracker templates in `../trackers`.
- Keep global planning and governance in `../program_documents`.
- Link sprint work to backlog items, requirements, releases, risks, decisions, and learnings.

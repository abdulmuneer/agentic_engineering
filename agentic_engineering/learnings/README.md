# Learnings Folder

This folder stores important discoveries made during execution.

Use it for implementation lessons, unexpected constraints, bug patterns, operational findings, security lessons, user feedback themes, and process improvements that should influence future work.

## What Belongs Here

- Technical discoveries.
- Repeated mistakes or rework causes.
- Production incidents and postmortems.
- Useful debugging notes.
- New domain insights.
- Lessons from reviews, testing, support, and retrospectives.

## Rules

- Keep entries specific and actionable.
- Link back to related requirements, decisions, defects, releases, or incidents.
- Convert repeatable lessons into checklists, tests, evals, skills, documentation, permission rules, runbooks, or backlog items.
- Link lessons from agentic work to the relevant work packet.
- Promote repeated successful workflows to `../agentic/skill_registry.md`.
- Promote repeated failures or review findings to `../agentic/eval_registry.md` or tests.

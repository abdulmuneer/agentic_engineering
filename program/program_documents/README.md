# Program Documents

This folder stores general program documentation that is broader than a single sprint.

Use it for global planning, program chartering, governance, stakeholder maps, operating assumptions, strategic context, delivery principles, and other documents that should guide the whole program.

## Suggested Documents

| File | Purpose |
|---|---|
| `program_charter_template.md` | Defines why the program exists, who owns it, and what success means. |
| `global_plan_template.md` | Captures broad planning across milestones, releases, teams, and major dependencies. |
| `governance_model_template.md` | Defines decision rights, approval gates, escalation paths, and review cadence. |

## Rules

- Keep sprint-specific plans in `../sprints`, not here.
- Link general program documents to trackers when they affect scope, priority, risk, or decisions.
- Update these documents when the program direction changes materially.
